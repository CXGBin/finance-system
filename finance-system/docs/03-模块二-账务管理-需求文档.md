# 模块二：账务管理 — 详细功能需求文档

> 版本：v1.0 | 日期：2026-06-15 | 状态：待确认  
> 模块标识：`account` | 默认状态：开启

---

## 1. 模块定位

账务管理是财务管理系统的核心业务模块，负责会计科目体系维护、会计凭证的完整生命周期管理（录入→审核→记账→结账）、账簿查询以及期末处理。报表模块直接依赖本模块产生的凭证数据生成各类报表。

## 3. 详细功能说明

### A-01 会计科目管理

#### 业务规则
- 科目编码规则：一级 4 位（1001、1002），二级 2 位（1001.01），最多支持 6 级
- 科目层级由编码规则自动确定，父子关系不可手动调整
- 预置标准会计科目模板（参考 CAS 会计科目体系）
- 科目类型：资产(1)、负债(2)、权益(3)、收入(4)、费用(5)
- 余额方向：资产/费用 → 借方；负债/权益/收入 → 贷方
- 已使用科目（有发生额或余额）不可删除，仅可停用
- 一级科目编码不可修改，下级科目编码新增后不可修改
- 支持科目批量导入（Excel 模板）

#### 科目字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| subject_code | string | 是 | 科目编码，如 `1001`、`1002.01` |
| subject_name | string | 是 | 科目名称 |
| parent_id | long | 否 | 父级科目ID（根据编码自动确定） |
| subject_level | int | 自动 | 层级（编码中 `.` 的数量+1） |
| subject_type | int | 是 | 1资产 2负债 3权益 4收入 5费用 |
| balance_direction | int | 是 | 1借方 2贷方 |
| is_enabled | int | 是 | 0停用 1启用 |
| is_cash | int | 否 | 是否现金/银行科目（用于日记账） |
| is_bank | int | 否 | 是否银行科目 |
| auxiliary_type | string | 否 | 辅助核算类型：空/department/project/customer |
| sort_order | int | 否 | 同级排序 |
| remark | string | 否 | 备注 |

#### 查询条件
- 按科目编码/名称模糊搜索
- 按科目类型筛选
- 按状态筛选（启用/全部）

#### 接口
- `GET /api/account/subject/tree` — 科目树（启用状态）
- `GET /api/account/subject/tree-all` — 科目树（含停用）
- `GET /api/account/subject/{id}` — 科目详情
- `POST /api/account/subject` — 新增科目
- `PUT /api/account/subject/{id}` — 修改科目
- `DELETE /api/account/subject/{id}` — 删除（校验无发生额）
- `PUT /api/account/subject/{id}/status` — 启用/停用
- `POST /api/account/subject/import` — Excel 批量导入
- `GET /api/account/subject/export` — Excel 导出

### A-02 科目期初余额

#### 业务规则
- 在启用系统或新会计年度开始时录入各科目期初余额
- 仅需录入末级科目（最底层）的余额，上级科目自动汇总
- 资产/费用类科目录入借方余额，负债/权益/收入类录入贷方余额
- 录入完成后进行试算平衡校验：所有科目借方余额合计 = 贷方余额合计
- 试算不平衡时不允许保存
- 第一个会计期间录入后，后续期间自动继承上期期末余额作为本期期初
- 已录入凭证的科目不允许修改期初余额（需先删除凭证）

#### 余额字段

| 字段 | 类型 | 说明 |
|------|------|------|
| subject_id | long | 科目ID |
| period_id | long | 会计期间ID |
| begin_debit | decimal(18,2) | 期初借方余额 |
| begin_credit | decimal(18,2) | 期初贷方余额 |

#### 接口
- `GET /api/account/subject/balance/{periodId}` — 查询某期间期初余额
- `POST /api/account/subject/balance` — 批量保存期初余额
- `GET /api/account/subject/balance/trial-balance` — 试算平衡校验

### A-03 凭证录入

#### 业务规则
- 每张凭证必须至少包含 2 条分录（一借一贷以上）
- 借方金额合计必须等于贷方金额合计，否则不允许保存
- 凭证日期必须在当前未关闭的会计期间内
- 凭证编号支持自动生成（按期间递增）和手工编号两种模式
- 凭证类型：收款(1)、付款(2)、转账(3)
- 凭证摘要可按科目自动带出常用摘要（可选）
- 分录行数不限，支持动态增减行
- 科目只能选择末级科目
- 支持从模板快速创建凭证（可选）
- 保存后默认为「草稿」状态

#### 凭证字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| voucher_date | date | 是 | 凭证日期 |
| period_id | long | 自动 | 会计期间ID（根据日期自动确定） |
| voucher_type | int | 是 | 1收款 2付款 3转账 |
| voucher_no | string | 自动 | 凭证编号（自动/手动） |
| abstract_text | string | 否 | 凭证总摘要 |

#### 分录字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| summary | string | 否 | 分录摘要 |
| subject_id | long | 是 | 科目ID（末级） |
| debit_amount | decimal(18,2) | 条件 | 借方金额 |
| credit_amount | decimal(18,2) | 条件 | 贷方金额 |
| auxiliary_id | long | 条件 | 辅助核算ID（科目有辅助核算时必填） |

#### 接口
- `POST /api/account/voucher` — 新增凭证（草稿）

### A-04 凭证查询

#### 业务规则
- 按多维度条件筛选凭证列表
- 支持查看凭证详情（含所有分录）
- 凭证状态过滤：全部/草稿/已审核/已作废
- 已审核凭证不可修改，可查看

#### 查询条件

| 条件 | 类型 | 说明 |
|------|------|------|
| voucher_no | string | 凭证号精确/模糊 |
| voucher_date_start | date | 起始日期 |
| voucher_date_end | date | 截止日期 |
| voucher_type | int | 凭证类型 |
| status | int | 状态 |
| subject_id | long | 包含某科目的凭证 |
| keyword | string | 摘要关键词 |
| prepared_by | long | 制单人 |
| reviewed_by | long | 审核人 |

#### 接口
- `GET /api/account/voucher/page` — 分页查询
- `GET /api/account/voucher/{id}` — 凭证详情

### A-05 凭证修改

#### 业务规则
- 仅「草稿」状态的凭证可修改
- 修改后重新校验借贷平衡
- 修改后状态仍为草稿

#### 接口
- `PUT /api/account/voucher/{id}` — 修改凭证

---

### A-06 凭证审核

#### 业务规则
- 制单人与审核人不可为同一人（四眼原则）
- 审核后凭证状态变为「已审核」，不可修改和删除
- 支持反审核（需有反审核权限）
- 反审核前提：凭证所在会计期间未结账
- 反审核后凭证恢复为「草稿」状态

#### 接口
- `POST /api/account/voucher/{id}/audit` — 审核
- `POST /api/account/voucher/{id}/unaudit` — 反审核

---

### A-07 凭证作废

#### 业务规则
- 已审核和草稿凭证均可作废
- 作废后凭证内容保留，标记为「已作废」
- 作废凭证不参与账簿计算和报表汇总
- 作废操作不可逆

#### 接口
- `POST /api/account/voucher/{id}/void` — 作废

---

### A-08 凭证打印

#### 业务规则
- 生成标准记账凭证格式 PDF
- 支持批量打印（选定多张凭证）
- 打印格式遵循标准记账凭证样式

#### 接口
- `GET /api/account/voucher/{id}/print` — 打印单张
- `POST /api/account/voucher/print-batch` — 批量打印

### A-14 会计期间管理

#### 业务规则
- 会计期间按年度+月份管理（如 2026-01、2026-02 ... 2026-12）
- 每个期间有明确的起止日期
- 期间状态：未开始、进行中、已结账
- 同一时间只能有一个期间处于「进行中」状态
- 已结账期间不可录入凭证
- 支持反结账（需有反结账权限，且下一期间未结账）
- 年末结账时自动创建下一年度的12个期间

#### 期间字段

| 字段 | 类型 | 说明 |
|------|------|------|
| period_year | int | 年度 |
| period_month | int | 月份 |
| begin_date | date | 开始日期 |
| end_date | date | 结束日期 |
| is_closed | bool | 是否已结账 |
| closed_time | datetime | 结账时间 |
| closed_by | long | 结账人 |

#### 接口
- `GET /api/account/period/list` — 期间列表（按年度）
- `POST /api/account/period/init-year` — 初始化年度期间
- `GET /api/account/period/current` — 获取当前期间

### A-15 期末损益结转

#### 业务规则
- 自动将所有损益类科目（收入/费用）的期末余额结转到「本年利润」科目
- 生成一张转账凭证，借记收入类科目、贷记费用类科目，差额进入本年利润
- 结转前检查：当期所有凭证已审核
- 结转生成的凭证自动标记为「已审核」
- 重复结转时提示「已结转」

#### 接口
- `POST /api/account/period/profit-transfer` — 损益结转

---

### A-16 期末结账

#### 业务规则
- 结账前检查：
  - 当期所有凭证已审核
  - 损益结转已完成
  - 试算平衡（借方合计=贷方合计）
  - 当前期间无未完成的期末处理
- 结账后当期不可录入新凭证
- 年末（12月）结账后自动结转本年利润到利润分配
- 结账成功后自动将下一期间设为「进行中」

#### 接口
- `POST /api/account/period/close` — 期末结账
- `POST /api/account/period/unclose` — 反结账

---

### A-17 辅助核算

#### 业务规则
- 在科目上配置辅助核算类型（部门/项目/往来单位/自定义）
- 凭证录入时，选用了辅助核算的科目需额外选择辅助核算项
- 账簿查询支持按辅助核算维度汇总
- 辅助核算项管理：

| 辅助类型 | 管理表 | 说明 |
|---------|--------|------|
| department | sys_dept | 复用部门管理 |
| project | fm_aux_project | 项目核算 |
| customer | fm_aux_customer | 往来单位（客户） |
| supplier | fm_aux_supplier | 往来单位（供应商） |

#### 接口
- `GET /api/account/auxiliary/{type}/list` — 辅助核算项列表
- `POST /api/account/auxiliary/{type}` — 新增辅助项
- `PUT /api/account/auxiliary/{type}/{id}` — 修改
- `DELETE /api/account/auxiliary/{type}/{id}` — 删除
- `GET /api/account/ledger/auxiliary-detail` — 辅助核算明细账

## 6. 数据库补充表设计

### fm_aux_project（项目辅助核算表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| project_code | varchar(50) UNIQUE | 项目编码 |
| project_name | varchar(100) | 项目名称 |
| manager | string | 项目负责人 |
| begin_date | date | 开始日期 |
| end_date | date | 结束日期 |
