# 模块四：预算管理 — 详细功能需求文档

> 版本：v1.0 | 日期：2026-06-15 | 状态：待确认  
> 模块标识：`budget` | 默认状态：开启  
> 依赖模块：`account`（账务管理）、`system`（系统管理）

---

## 1. 模块定位

预算管理模块提供企业预算的编制、执行跟踪、分析和调整能力。通过与账务模块的凭证数据联动，实时展示预算执行进度和偏差分析，帮助企业控制成本、合理分配资源。可开启审批流程模块实现预算审批。

---

## 3. 详细功能说明

### B-01 预算项目设置

#### 业务规则
- 预算可按以下维度编制：
  - **科目维度**：关联会计科目（费用类为主）
  - **部门维度**：关联组织架构中的部门
  - **期间维度**：年度预算 / 季度预算 / 月度预算
- 预算科目可从会计科目中选择（通常为费用类科目）
- 支持按「预算年度」管理，每个年度独立
- 预算状态：草稿 / 审批中 / 已批准 / 执行中 / 已冻结

#### 预算年度字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | int | 是 | 预算年度 |
| status | int | 是 | 0草稿 1审批中 2已批准 3执行中 4已冻结 |
| description | string | 否 | 说明 |
| created_by | bigint FK | 创建人 |
| created_time | datetime | 自动 | 创建时间 |

#### 预算科目字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| budget_year_id | bigint FK | 是 | 预算年度ID |
| subject_id | bigint FK | 是 | 关联会计科目 |
| dept_id | bigint FK | 否 | 关联部门（为空表示全公司级别） |
| annual_amount | decimal(18,2) | 是 | 年度预算总额 |
| remark | string | 否 | 备注 |

#### 接口
- `GET /api/budget/setting/years` — 预算年度列表
- `POST /api/budget/setting/year` — 创建预算年度
- `PUT /api/budget/setting/year/{id}/status` — 更新年度状态
- `GET /api/budget/setting/subjects/{yearId}` — 预算科目列表
- `POST /api/budget/setting/subject` — 新增预算科目
- `PUT /api/budget/setting/subject/{id}` — 修改
- `DELETE /api/budget/setting/subject/{id}` — 删除

---

### B-02 预算编制

#### 业务规则
- 在预算年度下按科目+部门编制预算
- 年度预算金额为必填，可按月度拆分（手动/自动平均）
- 月度拆分后，12个月之和必须等于年度预算总额
- 支持模板导入（Excel 模板批量导入预算数据）
- 编制完成后提交审批（若审批模块开启则走审批流，否则直接标记为已批准）

#### 月度预算字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| budget_subject_id | bigint FK | 是 | 预算科目ID |
| month | int | 是 | 月份(1-12) |
| amount | decimal(18,2) | 是 | 月度预算金额 |

#### 接口
- `GET /api/budget/plan/{yearId}` — 查看某年度预算编制
- `POST /api/budget/plan/monthly` — 保存月度预算
- `POST /api/budget/plan/auto-split` — 自动平均拆分年度预算到月度
- `POST /api/budget/plan/import` — Excel 批量导入
- `GET /api/budget/plan/export` — Excel 导出

---

### B-03 预算审批

#### 业务规则
- 当审批流程模块开启时，预算编制完成后需发起审批
- 审批通过后预算状态变为「已批准」
- 审批流程：编制人提交 → 部门负责人审批 → 财务主管审批
- 当审批模块未开启时，具有权限的管理员可直接标记为已批准

#### 接口
- `POST /api/budget/plan/{yearId}/submit` — 提交预算审批
- `POST /api/budget/plan/{yearId}/approve` — 审批通过
- `POST /api/budget/plan/{yearId}/reject` — 审批驳回

---

### B-04 预算执行跟踪

#### 业务规则
- 实时从账务模块获取凭证数据，按科目+部门汇总本期实际发生额
- 计算预算执行率：`实际发生额 / 预算金额 × 100%`
- 展示每个预算科目的：
  - 年度预算总额
  - 已执行金额（累计实际发生额）
  - 执行率
  - 剩余预算
- 支持按月度查看执行明细
- 数据实时计算，无需手动刷新（每次查询时实时汇总）

#### 查询条件

| 条件 | 类型 | 说明 |
|------|------|------|
| budget_year_id | long | 预算年度 |
| month | int | 月份（可选，筛选某月） |
| dept_id | long | 部门（可选） |
| subject_id | long | 科目（可选） |
| execution_rate_min | decimal | 最低执行率（可选） |
| execution_rate_max | decimal | 最高执行率（可选） |

#### 输出字段

| 字段 | 类型 | 说明 |
|------|------|------|
| subject_code | string | 科目编码 |
| subject_name | string | 科目名称 |
| dept_name | string | 部门名称 |
| annual_budget | decimal | 年度预算 |
| executed_amount | decimal | 已执行金额 |
| remaining_budget | decimal | 剩余预算 |
| execution_rate | decimal | 执行率(%) |
| month_budget | decimal | 月度预算 |
| month_executed | decimal | 月度已执行 |
| month_remaining | decimal | 月度剩余 |

#### 接口
- `GET /api/budget/execution` — 预算执行查询

---

### B-05 预算预警

#### 业务规则
- 预警阈值可配置：
  - **黄色预警**：执行率 ≥ 80%（默认）
  - **红色预警**：执行率 ≥ 100%（超支）
- 预警触发后：系统内消息通知 + 页面预警标识
- 管理员可在预警列表中查看所有触发的预警
- 支持按「已处理 / 未处理」筛选预警记录

#### 预警配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| yellow_threshold | 80 | 黄色预警执行率阈值(%) |
| red_threshold | 100 | 红色预警执行率阈值(%) |
| notify_enabled | true | 是否发送通知 |
| notify_roles | string[] | 接收通知的角色 |

#### 接口
- `GET /api/budget/alert/config` — 获取预警配置
- `PUT /api/budget/alert/config` — 修改预警配置
- `GET /api/budget/alert/list` — 预警记录列表
- `PUT /api/budget/alert/{id}/handle` — 标记预警已处理

---

### B-06 预算分析

#### 业务规则
- 提供多维度预算分析报表：
  - **预算与实际对比**：按科目/部门展示预算与实际金额、差异额、差异率
  - **趋势分析**：月度预算执行趋势折线图
  - **部门排名**：各部门预算执行率排名
  - **科目排名**：费用类科目支出排名（TOP 10）
- 支持图表展示（柱状图、折线图、饼图）

#### 分析类型

| 分析维度 | 查询条件 | 图表类型 |
|---------|---------|---------|
| 科目对比 | 年度+部门 | 柱状图（预算 vs 实际） |
| 月度趋势 | 年度+科目 | 折线图 |
| 部门执行率 | 年度 | 水平柱状图 |
| 费用 TOP10 | 年度+月度 | 饼图 |

#### 接口
- `GET /api/budget/analysis/subject-compare` — 科目对比分析
- `GET /api/budget/analysis/monthly-trend` — 月度趋势分析
- `GET /api/budget/analysis/dept-ranking` — 部门排名
- `GET /api/budget/analysis/top-expense` — 费用 TOP10

---

### B-07 预算调整

#### 业务规则
- 支持对已批准的预算进行调整（追加/调减）
- 调整需要填写调整原因
- 单次调整金额不可超过年度预算的 50%（可配置）
- 调整需审批（审批模块开启时）
- 调整后预算金额 = 原预算 + 调整金额（追加为正，调减为负）
- 保留完整的调整历史记录

#### 调整字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| budget_subject_id | bigint FK | 是 | 预算科目ID |
| adjust_type | int | 是 | 1追加 2调减 |
| adjust_amount | decimal(18,2) | 是 | 调整金额（正数） |
| reason | string | 是 | 调整原因 |
| status | int | 是 | 0待审批 1已批准 2已驳回 |

#### 接口
- `GET /api/budget/adjust/page` — 调整记录分页
- `POST /api/budget/adjust` — 提交调整申请
- `POST /api/budget/adjust/{id}/approve` — 审批通过
- `POST /api/budget/adjust/{id}/reject` — 审批驳回
- `GET /api/budget/adjust/history/{budgetSubjectId}` — 某科目调整历史

---

### B-08 预算报表导出

#### 业务规则
- 支持预算执行表、预算对比分析表导出
- 格式：Excel (.xlsx)、PDF

#### 接口
- `GET /api/budget/export/execution` — 导出预算执行表
- `GET /api/budget/export/analysis` — 导出预算分析报表
