# 模块一：系统管理 — 详细功能需求文档

> 版本：v1.0 | 日期：2026-06-15 | 状态：待确认  
> 模块标识：`system` | 默认状态：**核心模块，始终开启**

---

## 1. 模块定位

系统管理是整个财务管理系统的核心基础模块，负责用户认证授权、组织架构、菜单权限、数据字典、操作审计、模块开关管理等基础能力。该模块**不可关闭**，其他所有业务模块均依赖本模块提供的能力。

## 3. 详细功能说明

### S-01 用户登录/登出

#### 业务规则
- 支持用户名 + 密码登录
- 密码使用 BCrypt 加密存储，不可逆
- 登录成功返回 `access_token`（有效期 2 小时）和 `refresh_token`（有效期 7 天）
- 连续 5 次密码错误，锁定账户 30 分钟
- 支持「记住我」选项：勾选后 refresh_token 有效期延长至 30 天
- 登出时使 Token 失效（Redis 黑名单）

#### 输入字段

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| username | string | 是 | 长度 3-50，字母/数字/下划线 |
| password | string | 是 | 长度 6-32 |
| rememberMe | boolean | 否 | true/false |

#### 输出字段

| 字段 | 类型 | 说明 |
|------|------|------|
| access_token | string | JWT Token |
| refresh_token | string | 刷新 Token |
| expires_in | int | Token 有效期（秒） |
| user_info | object | 用户基本信息（id、姓名、角色列表、权限列表、模块列表） |

#### 接口
- `POST /api/auth/login`
- `POST /api/auth/logout`

---

### S-02 Token 刷新

#### 业务规则
- 当 access_token 过期时，前端自动使用 refresh_token 换取新 token
- refresh_token 只能使用一次，换取后旧 token 立即失效
- refresh_token 过期需重新登录

#### 输入字段

| 字段 | 类型 | 必填 |
|------|------|------|
| refresh_token | string | 是 |

#### 接口
- `POST /api/auth/refresh-token`

---

### S-03 修改密码

#### 业务规则
- 用户自行修改，需验证旧密码
- 新密码不可与最近 3 次密码相同（可选）
- 修改成功后当前 Token 失效，需重新登录

#### 输入字段

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| old_password | string | 是 | 非空 |
| new_password | string | 是 | 6-32 位，含字母和数字 |
| confirm_password | string | 是 | 与 new_password 一致 |

#### 接口
- `PUT /api/system/user/password`

---

### S-04 用户管理

#### 业务规则
- 管理员可进行用户 CRUD
- 用户名唯一，创建后不可修改
- 支持分配角色（多角色）
- 支持分配部门
- 用户状态：启用 / 停用
- 管理员可重置用户密码为默认密码（如 `123456`）
- 不支持删除已有关联数据的用户（凭证、日志等），仅支持停用
- 初始超级管理员 `admin` 不可删除、不可停用

#### 列表查询条件

| 查询条件 | 类型 | 说明 |
|---------|------|------|
| username | string | 模糊搜索 |
| real_name | string | 模糊搜索 |
| phone | string | 精确/模糊 |
| status | int | 0/1 筛选 |
| dept_id | long | 部门筛选 |
| role_id | long | 角色筛选 |
| createTime | date range | 创建时间范围 |

#### 新增/编辑字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 创建时必填，编辑时不可改 |
| password | string | 创建时必填 | 编辑时为空表示不修改 |
| real_name | string | 是 | 真实姓名 |
| email | string | 否 | 邮箱格式校验 |
| phone | string | 否 | 手机号格式校验 |
| dept_id | long | 是 | 所属部门 |
| role_ids | long[] | 是 | 角色列表 |
| status | int | 是 | 0停用 1启用 |
| avatar | string | 否 | 头像 URL |
| remark | string | 否 | 备注 |

#### 接口
- `GET /api/system/user/page` — 分页查询
- `GET /api/system/user/{id}` — 查询详情
- `POST /api/system/user` — 新增
- `PUT /api/system/user/{id}` — 修改
- `DELETE /api/system/user/{id}` — 删除（校验关联）
- `PUT /api/system/user/{id}/status` — 切换状态
- `PUT /api/system/user/{id}/reset-password` — 重置密码

### S-05 角色管理

#### 业务规则
- 角色编码全局唯一
- 支持为角色分配菜单权限（树形勾选）
- 角色状态：启用 / 停用
- 停用的角色，其用户登录后不拥有对应权限
- 预置角色：超级管理员（不可删除）、财务主管、会计、出纳、普通员工

#### 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| role_name | string | 是 | 角色名称 |
| role_code | string | 是 | 角色编码（大写字母+下划线） |
| description | string | 否 | 描述 |
| sort_order | int | 否 | 排序号 |
| status | int | 是 | 0停用 1启用 |
| menu_ids | long[] | 否 | 菜单权限 ID 列表（编辑时传入） |

#### 接口
- `GET /api/system/role/page` — 分页查询
- `GET /api/system/role/list` — 下拉列表（不分页）
- `GET /api/system/role/{id}` — 查询详情
- `POST /api/system/role` — 新增
- `PUT /api/system/role/{id}` — 修改
- `DELETE /api/system/role/{id}` — 删除（校验是否有关联用户）
- `GET /api/system/role/{id}/menus` — 获取角色已分配菜单

### S-06 菜单管理

#### 业务规则
- 菜单为树形结构（最多 4 级）
- 菜单类型：目录（1）、菜单（2）、按钮/权限（3）
- 目录：仅用于分组，不可点击
- 菜单：对应前端页面，需配置路由路径和组件路径
- 按钮：对应页面上的操作按钮（新增/编辑/删除/审核等），仅作为权限标识
- 每个菜单需关联模块标识（moduleId），关闭模块时自动隐藏对应菜单
- 预置系统管理菜单数据（初始化脚本）

#### 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| parent_id | long | 否 | 父级ID，0为顶级 |
| menu_name | string | 是 | 菜单名称 |
| menu_type | int | 是 | 1目录 2菜单 3按钮 |
| path | string | 条件 | 菜单类型为1/2时必填 |
| component | string | 条件 | 菜单类型为2时必填 |
| permission | string | 条件 | 菜单类型为3时必填，如 `account:voucher:add` |
| icon | string | 否 | 菜单图标（antd 图标名） |
| module_id | string | 否 | 所属模块标识 |
| sort_order | int | 否 | 排序号（同级内） |
| visible | int | 否 | 0隐藏 1显示，默认1 |
| status | int | 是 | 0停用 1启用 |

#### 接口
- `GET /api/system/menu/tree` — 获取完整菜单树
- `POST /api/system/menu` — 新增菜单
- `PUT /api/system/menu/{id}` — 修改菜单
- `DELETE /api/system/menu/{id}` — 删除菜单（级联删除子菜单）

### S-07 部门管理

#### 业务规则
- 部门为树形结构（无层级限制）
- 每个部门有负责人
- 部门状态：启用 / 停用
- 停用部门后，其下的用户不可登录（或给出警告）

#### 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| parent_id | long | 否 | 父级ID，0为顶级 |
| dept_name | string | 是 | 部门名称 |
| sort_order | int | 否 | 排序 |
| leader | string | 否 | 负责人姓名 |
| phone | string | 否 | 联系电话 |
| email | string | 否 | 部门邮箱 |
| status | int | 是 | 0停用 1启用 |

#### 接口
- `GET /api/system/dept/tree` — 部门树
- `POST /api/system/dept` — 新增
- `PUT /api/system/dept/{id}` — 修改
- `DELETE /api/system/dept/{id}` — 删除（校验是否有关联用户）

#### 业务规则
- 岗位从属于某个部门
- 一个用户可关联一个岗位
- 岗位信息用于审批流程中的角色匹配

#### 字段定义

| 字段 | 类型 | 必填 | 说明 |

| dept_id | long | 是 | 所属部门 |
| post_name | string | 是 | 岗位名称 |

| sort_order | int | 否 | 排序 |
| status | int | 是 | 0停用 1启用 |

#### 接口
- `GET /api/system/post/page` — 分页查询（支持按部门筛选）
- `POST /api/system/post` — 新增
- `PUT /api/system/post/{id}` — 修改
- `DELETE /api/system/post/{id}` — 删除

### S-09 数据字典

#### 业务规则
- 字典按「字典类型」分组，每个类型下有多个「字典项」
- 字典类型编码唯一
- 字典用于系统各处下拉框、标签等
- 预置字典：凭证类型、科目类型、余额方向、性别、状态等
- 支持启用/停用单个字典项

#### 字典类型字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dict_name | string | 是 | 字典名称（如"凭证类型"） |
| dict_type | string | 是 | 字典编码（如"voucher_type"），唯一 |
| status | int | 是 | 0停用 1启用 |
| remark | string | 否 | 备注 |

#### 字典项字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dict_type | string | 是 | 所属字典类型编码 |
| dict_label | string | 是 | 显示值 |
| dict_value | string | 是 | 实际值 |
| sort_order | int | 否 | 排序 |
| status | int | 是 | 0停用 1启用 |

#### 接口
- `GET /api/system/dict/type/page` — 字典类型分页
- `POST /api/system/dict/type` — 新增字典类型
- `PUT /api/system/dict/type/{id}` — 修改
- `DELETE /api/system/dict/type/{id}` — 删除
- `GET /api/system/dict/data/{dictType}` — 获取某类型下所有字典项
- `POST /api/system/dict/data` — 新增字典项
- `PUT /api/system/dict/data/{id}` — 修改字典项
- `DELETE /api/system/dict/data/{id}` — 删除字典项

### S-10 操作日志

#### 业务规则
- 自动记录用户的关键操作（增/删/改）
- 记录内容：操作人、模块、操作类型、请求参数、IP 地址、耗时
- 日志仅查询，不可修改和删除
- 支持按模块、操作人、时间范围、操作类型筛选
- 日志保留周期：90 天（可配置），过期自动清理

#### 字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | long | 操作人ID |
| user_name | string | 操作人姓名 |
| module | string | 所属模块 |
| action | string | 操作类型（ADD/UPDATE/DELETE/LOGIN/LOGOUT） |
| description | string | 操作描述 |
| ip_address | string | IP 地址 |
| request_url | string | 请求 URL |
| request_method | string | HTTP 方法 |
| request_body | text | 请求参数（JSON） |
| response_code | int | 响应码 |
| duration_ms | int | 接口耗时(ms) |
| created_time | datetime | 操作时间 |

#### 查询条件

| 条件 | 类型 | 说明 |
|------|------|------|
| module | string | 模块筛选 |
| action | string | 操作类型筛选 |
| user_name | string | 操作人模糊搜索 |
| start_time | date | 开始时间 |
| end_time | date | 结束时间 |

#### 接口
- `GET /api/system/log/page` — 分页查询
- `GET /api/system/log/{id}` — 查看详情（含请求体）

---

### S-11 模块管理

#### 业务规则
- 列出系统所有模块及其状态
- 核心模块（system）的开关不可更改
- 关闭模块时校验：如果其他已开启模块依赖此模块，则提示"请先关闭依赖模块"
- 开关变更需超级管理员权限
- 变更后实时生效（前端侧边栏菜单、后端 API 拦截同步更新）

#### 列表字段

| 字段 | 类型 | 说明 |
|------|------|------|
| module_id | string | 模块标识 |
| module_name | string | 模块名称 |
| description | string | 描述 |
| is_enabled | boolean | 是否启用 |
| is_core | boolean | 是否核心模块 |
| dependencies | string[] | 依赖模块 |
| sort_order | int | 排序 |

#### 接口
- `GET /api/system/module/list` — 获取模块列表
- `PUT /api/system/module/{moduleId}/toggle` — 切换模块开关（true/false）
- `GET /api/system/module/{moduleId}/dependencies` — 获取模块依赖关系

## 4. 页面设计要求

### 4.1 页面列表

| 页面 | 路由 | 说明 |
|------|------|------|
| 登录页 | `/login` | 独立页面，居中卡片式 |
| 用户管理 | `/system/user` | 表格 + 搜索 + 弹窗表单 |
| 角色管理 | `/system/role` | 表格 + 搜索 + 弹窗表单（含菜单权限树） |
| 菜单管理 | `/system/menu` | 左侧树 + 右侧表单 |
| 部门管理 | `/system/dept` | 左侧树 + 右侧列表 |
| 岗位管理 | `/system/post` | 表格 + 搜索 + 弹窗表单 |
| 字典管理 | `/system/dict` | 左侧字典类型 + 右侧字典项 |
| 操作日志 | `/system/log` | 表格 + 搜索（不可编辑） |
| 模块管理 | `/system/module` | 卡片列表 + 开关 Switch |
| 系统配置 | `/system/config` | 分组表单（Tab 页签分类） |
| 个人中心 | `/system/profile` | 个人信息编辑页 |

### 4.2 UI 规范
- 所有列表页顶部包含搜索栏（可折叠）
- 新增/编辑使用 Drawer 或 Modal 弹窗
- 树形数据使用 antd Tree/TreeSelect 组件
- 操作列固定在右侧，支持 dropdown 更多操作
- 分页使用 antd Pagination 组件
- 删除操作需二次确认弹窗

### sys_dict_data（字典数据表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| dict_type | varchar(100) | 所属字典编码 |
| dict_label | varchar(100) | 显示值 |
| dict_value | varchar(100) | 实际值 |
| sort_order | int | 排序 |
| status | tinyint | 状态 |
| created_time | datetime | 创建时间 |

### sys_config（系统配置表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| config_key | varchar(100) UNIQUE | 配置键 |
| config_value | varchar(500) | 配置值 |
| config_group | varchar(50) | 配置分组 |
| description | varchar(200) | 描述 |
| updated_time | datetime | 更新时间 |
