# 模块六：资产管理 — 详细功能需求文档

> 版本：v1.0 | 日期：2026-06-15 | 状态：待确认  
> 模块标识：`asset` | 默认状态：开启  
> 依赖模块：`account`（账务管理，折旧凭证自动生成）

---

## 1. 模块定位

资产管理模块负责固定资产的完整生命周期管理——从资产登记、折旧计提到资产变动（调拨、处置、报废）和资产盘点。折旧数据可自动生成会计凭证，与账务模块联动。

## 3. 详细功能说明

### AS-01 资产分类管理

#### 业务规则
- 资产分类为树形结构（如：房屋及构筑物 → 办公楼）
- 每个分类关联默认折旧方法、使用年限、残值率
- 新增资产时选择分类，自动继承默认折旧参数
- 已使用的分类不可删除

#### 分类字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| parent_id | long | 否 | 父级ID |
| category_name | string | 是 | 分类名称 |
| category_code | string | 是 | 分类编码 |
| depreciation_method | int | 是 | 1直线法 2双倍余额递减法 3年数总和法 |
| useful_life_months | int | 是 | 默认使用年限（月） |
| residual_rate | decimal(5,2) | 是 | 默认残值率(%)，如 5.00 |
| status | tinyint | 是 | 0停用 1启用 |

#### 接口
- `GET /api/asset/category/tree` — 分类树
- `POST /api/asset/category` — 新增
- `PUT /api/asset/category/{id}` — 修改
- `DELETE /api/asset/category/{id}` — 删除

### AS-02 资产卡片管理

#### 业务规则
- 每个固定资产对应一张资产卡片
- 资产编号自动生成（分类编码+年月+流水号）
- 资产状态：在用(1)、闲置(2)、维修中(3)、已处置(4)、已报废(5)
- 支持上传资产图片
- 已计提折旧的资产卡片关键信息（原值、年限）不可修改
- 支持资产卡片批量导入

### AS-03 资产折旧管理

#### 业务规则
- 每月可执行一次折旧计提操作
- 折旧范围：所有状态为「在用」的资产卡片
- 折旧公式：
  - **直线法**：`月折旧额 = (原值 - 残值) / 使用月数`
  - **双倍余额递减法**：`月折旧额 = 期初净值 × (2 / 使用月数)`，最后两年转为直线法
  - **年数总和法**：`月折旧额 = (原值 - 残值) × (剩余月数 / 总月数序和)`
- 当月新入账资产次月开始计提折旧
- 已提足折旧的资产停止计提（累计折旧 ≥ 原值-残值）
- 折旧计提后可自动生成会计凭证（借：管理费用/制造费用，贷：累计折旧）
- 同一期间不可重复计提折旧
- 支持查看单资产的折旧明细历史

#### 接口
- `POST /api/asset/depreciation/calculate` — 执行月度折旧计提
- `POST /api/asset/depreciation/generate-voucher` — 生成折旧凭证
- `GET /api/asset/depreciation/preview/{period}` — 预览某月折旧数据
- `GET /api/asset/depreciation/history/{assetId}` — 单资产折旧明细

### AS-04 资产变动管理

#### 业务规则
- 变动类型：部门调拨(1)、资产处置(2)、资产报废(3)、价值调整(4)
- **部门调拨**：变更资产使用部门
- **资产处置**：资产出售或捐赠，记录处置收入
- **资产报废**：资产报废，记录报废原因
- **价值调整**：资产评估增值/减值
- 变动审批：若审批模块开启则需审批
- 已处置/报废的资产停止折旧

#### 变动字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| asset_id | bigint FK | 是 | 资产ID |
| change_type | int | 是 | 1调拨 2处置 3报废 4价值调整 |
| target_dept_id | long | 条件 | 调拨目标部门 |
| disposal_income | decimal(18,2) | 条件 | 处置收入 |
| disposal_reason | string | 条件 | 处置/报废原因 |
| value_adjust | decimal(18,2) | 条件 | 价值调整金额 |
| apply_date | date | 是 | 变动日期 |
| status | int | 是 | 0待审批 1已审批 2已驳回 |

#### 接口
- `GET /api/asset/change/page` — 变动记录分页
- `POST /api/asset/change` — 提交变动申请
- `POST /api/asset/change/{id}/approve` — 审批通过
- `GET /api/asset/change/history/{assetId}` — 资产变动历史

---

### AS-05 资产盘点

#### 业务规则
- 可按部门/分类/全部发起盘点
- 生成盘点单，列出盘点范围内的所有资产
- 录入实际盘点结果：在用/闲置/丢失/损坏
- 系统自动对比账面数据与盘点结果，标记差异
- 差异处理：确认并更新资产状态
- 盘点完成后生成盘点报告

#### 盘点单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inventory_no | string | 自动 | 盘点单号 |
| inventory_name | string | 是 | 盘点名称 |
| scope_type | int | 是 | 1全部 2按部门 3按分类 |
| scope_ids | long[] | 条件 | 部门/分类ID列表 |
| status | int | 是 | 0待盘点 1盘点中 2已完成 |
| inventory_date | date | 是 | 盘点日期 |
| operator | bigint FK | 是 | 盘点人 |

#### 接口
- `GET /api/asset/inventory/page` — 盘点单分页
- `POST /api/asset/inventory` — 创建盘点单（自动生成盘点明细）
- `PUT /api/asset/inventory/{id}/start` — 开始盘点
- `PUT /api/asset/inventory/{id}/item/{assetId}` — 录入盘点结果
- `POST /api/asset/inventory/{id}/complete` — 完成盘点（生成差异报告）
- `GET /api/asset/inventory/{id}/report` — 查看盘点报告

### AS-06 资产报表

#### 业务规则
- **资产台账**：所有资产卡片列表汇总
- **折旧汇总表**：按月/按部门/按分类汇总折旧数据
- **资产增减变动表**：某期间内新增、处置、报废的资产统计
- **资产价值统计**：资产原值总额、累计折旧、净值
- 支持导出 Excel

#### 接口
- `GET /api/asset/report/ledger` — 资产台账
- `GET /api/asset/report/depreciation-summary` — 折旧汇总表
- `GET /api/asset/report/change-summary` — 资产增减变动表
- `GET /api/asset/report/value-statistics` — 资产价值统计
- `GET /api/asset/report/export` — 导出

---

### AS-07 折旧方式配置

#### 业务规则
- 系统预置三种折旧方法：直线法、双倍余额递减法、年数总和法
- 每种方法有固定的计算规则（代码实现）
- 不可自定义新增折旧方法（第一版）
