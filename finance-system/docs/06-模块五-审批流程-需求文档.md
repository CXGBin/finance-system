# 模块五：审批流程 — 详细功能需求文档

> 版本：v1.0 | 日期：2026-06-15 | 状态：待确认  
> 模块标识：`approval` | 默认状态：开启  
> 依赖模块：`system`（系统管理）

---

## 1. 模块定位

审批流程模块提供通用的审批能力，支持定义审批模板、配置审批节点、发起审批、处理审批任务。作为基础能力模块，可被预算管理、费用管理等业务模块调用，实现报销、采购、预算调整等业务单据的审批流程。

## 3. 详细功能说明

### AP-01 审批模板管理

#### 业务规则
- 模板定义一种审批类型（如「费用报销审批」「预算调整审批」）
- 每个模板包含多个审批节点，节点按顺序执行
- 审批节点类型：
  - **单人审批**：指定一人审批
  - **多人会签**：所有审批人都通过才通过
  - **多人或签**：任一审批人通过即可
  - **部门主管审批**：根据发起人所在部门自动匹配主管
  - **角色审批**：根据角色自动匹配审批人
- 审批人确定方式：
  - **指定人员**：模板中直接指定具体用户
  - **部门主管**：发起人直属部门的负责人
  - **角色匹配**：系统中拥有指定角色的用户
- 节点超时设置：超过指定天数未处理则触发超时提醒
- 模板状态：启用 / 停用

#### 模板字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| template_name | string | 是 | 模板名称 |
| template_code | string | 是 | 模板编码（唯一） |
| business_type | string | 是 | 业务类型（如 `expense_claim`） |
| description | string | 否 | 描述 |
| status | int | 是 | 0停用 1启用 |
| nodes | object[] | 是 | 审批节点定义（JSON） |

#### 审批节点定义

```json
{
  "nodes": [
    {
      "nodeIndex": 1,
      "nodeName": "部门主管审批",
      "nodeType": "single",        // single/multi_and/multi_or/dept_head/role
      "assigneeType": "dept_head", // specified/dept_head/role
      "assigneeValue": "",         // 指定人员时填写用户ID，角色时填写角色编码
      "timeoutDays": 3,            // 超时天数（0=不限）
      "rejectAction": "end",       // 驳回后动作：end(终止) / return(退回发起人) / previous(退回上一节点)
      "conditions": []             // 条件分支（可选，高级功能暂不实现）
    },
    {
      "nodeIndex": 2,
      "nodeName": "财务审批",
      "nodeType": "single",
      "assigneeType": "role",
      "assigneeValue": "FINANCE_MANAGER",
      "timeoutDays": 5,
      "rejectAction": "end"
    }
  ]
}
```

#### 接口
- `GET /api/approval/template/page` — 模板分页查询
- `GET /api/approval/template/list` — 模板下拉列表（启用状态）
- `GET /api/approval/template/{id}` — 模板详情
- `POST /api/approval/template` — 新增模板
- `PUT /api/approval/template/{id}` — 修改模板
- `DELETE /api/approval/template/{id}` — 删除（校验无进行中的审批）
- `PUT /api/approval/template/{id}/status` — 启用/停用

### AP-02 审批发起

#### 业务规则
- 审批发起时需关联业务单据（提供业务类型 + 业务单据ID）
- 系统根据业务类型自动匹配启用的审批模板
- 若无匹配模板则提示"未配置审批流程"
- 发起后自动创建审批实例和第一个节点的待办任务
- 审批发起人自动记录

#### 发起字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| template_id | long | 是 | 审批模板ID |
| business_type | string | 是 | 业务类型 |
| business_id | long | 是 | 业务单据ID |
| title | string | 是 | 审批标题 |
| description | string | 否 | 审批说明 |
| attachments | string[] | 否 | 附件URL列表 |

#### 接口
- `POST /api/approval/instance` — 发起审批
- `GET /api/approval/instance/available-templates` — 根据业务类型获取可用模板

---

### AP-03 待办任务

#### 业务规则
- 展示当前用户待处理的审批任务
- 显示：审批标题、发起人、发起时间、当前节点、待处理时长
- 支持按业务类型、发起时间筛选
- 超时未处理的任务标红提醒

#### 查询条件

| 条件 | 类型 | 说明 |
|------|------|------|
| business_type | string | 业务类型 |
| start_time | date | 发起时间起始 |
| end_time | date | 发起时间截止 |

#### 接口
- `GET /api/approval/pending/page` — 待办任务分页
- `GET /api/approval/pending/count` — 待办数量（用于首页徽标）

---

### AP-04 审批处理

#### 业务规则
- **通过**：当前节点通过，流转到下一节点；最后节点通过则审批完成
- **驳回**：根据模板配置的 `rejectAction` 执行（终止 / 退回发起人 / 退回上一节点）
- **转办**：将当前任务转交给其他人处理（转办人不改变后续流程）
- **加签**：在当前节点临时增加一个审批人
- 审批意见为必填项（通过时可选，驳回时必填）
- 审批操作记录不可撤回

#### 处理字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | string | 是 | approve / reject / transfer / countersign |
| comment | string | 条件 | 审批意见（驳回时必填） |
| transfer_to | long | 条件 | 转办目标用户ID（转办时必填） |
| countersign_user | long | 条件 | 加签用户ID（加签时必填） |

#### 接口
- `POST /api/approval/{taskId}/approve` — 通过
- `POST /api/approval/{taskId}/reject` — 驳回
- `POST /api/approval/{taskId}/transfer` — 转办
- `POST /api/approval/{taskId}/countersign` — 加签

---

### AP-05 已办任务

#### 业务规则
- 展示当前用户已处理过的审批任务
- 显示：审批标题、发起人、处理时间、处理结果（通过/驳回/转办）

#### 接口
- `GET /api/approval/done/page` — 已办任务分页

---

### AP-06 我的申请

#### 业务规则
- 展示当前用户发起的所有审批
- 显示：审批标题、当前状态（审批中/已通过/已驳回）、发起时间、当前节点

#### 接口
- `GET /api/approval/my/page` — 我的申请分页

---

### AP-07 审批详情

#### 业务规则
- 展示审批实例的完整信息：
  - 基本信息（标题、发起人、发起时间、审批模板）
  - 当前状态（审批中/已通过/已驳回）
  - 审批节点时间线（已完成/当前/待处理）
  - 每个节点的处理记录（审批人、处理时间、审批意见）
  - 关联的业务单据摘要信息

#### 接口
- `GET /api/approval/instance/{id}` — 审批详情

---

### AP-08 审批催办

#### 业务规则
- 仅审批发起人或管理员可催办
- 对当前待处理节点的审批人发送催办通知
- 同一节点每天最多催办 3 次
- 催办记录计入操作日志

#### 接口
- `POST /api/approval/instance/{id}/remind` — 发起催办

---

### AP-09 审批统计

#### 业务规则
- 统计维度：
  - 总审批数 / 通过数 / 驳回数 / 进行中数
  - 平均处理时长（按天）
  - 各业务类型的审批数量分布
- 支持按时间范围筛选
- 管理员可查看全公司统计，普通用户查看个人统计

#### 接口
- `GET /api/approval/statistics/overview` — 审批概览统计
- `GET /api/approval/statistics/trend` — 审批趋势（按月）

| 页面 | 路由 | 布局说明 |
|------|------|---------|
| 待办任务 | `/approval/pending` | 任务列表（高优先级置顶，超时标红） |

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |

| template_id | bigint FK | 审批模板ID |
| business_type | varchar(50) | 业务类型 |

| initiated_time | datetime | 发起时间 |
| finished_time | datetime | 完成时间 |

### fm_approval_task（审批任务表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
